# 第12课《找bug的小修理工》上集分镜稿

> **非常重要，必须严格遵守：以上传的图片作为关键帧画面；人物形象必须跟人物角色板保持一致，包括多视角、表情、服装、身体比例、局部细节和角色气质。不要生成新的角色和场景，都用我指定的参考图。**

## 基础设定

- 年龄：3-9岁
- 时长：约1分30秒
- 比例：16:9
- 主题：调试 = 找出并修复错误
- 核心知识：Bug是系统和程序里隐藏的小错误。调试就是顺着线索定位错误，并把它完美修复的闭环过程。调试四步骤（发现问题 ➔ 找到原因 ➔ 修改 ➔ 再试）与记录日志的重要性。
- 故事：团团和点点进入《找bug的小修理工》的情境，通过一个具体问题、一次动手观察和一次清楚总结，把本课知识变成小朋友能看懂、能复述的小故事。

## 角色与风格

- 团团：7岁男孩，黑灰短发，圆脸腮红，橙色上衣，蓝色背带裤，小书包，好奇活泼。
- 点点：白色云朵形AI机器人，深色屏幕脸，青蓝发光眼睛，胸口屏幕，头顶紫色天线。
- 本课新增角色/场景：工作室场景，工具台乱糟糟地摆着扳手、螺丝刀等工具，小自行车靠在墙边。
- 风格：儿童水彩绘本动画风，清透明亮，高清质感，色彩饱和，减少模糊笔触，角色与上传参考图一致。
- 负面：不要写实摄影、3D塑料感、暗黑恐怖、角色变形、多余手指、文字乱码、画面拥挤、低清晰度。
- 旁白要求：旁白讲话时，团团和点点的嘴不要动，不要做对口型或配合讲话动作；只保持倾听、观察或轻微点头即可。
- 配音要求：发音清晰，全部使用标准普通话。

# 第12课《找bug的小修理工》上集 画面核心故事脚本

## 基础设定

* **年龄**：3-9岁
* **总时长**：1分30秒（90秒）
* **比例**：16:9
* **主题**：调试 = 找出并修复错误
* **核心知识**：Bug是系统和程序里隐藏的小错误。调试就是顺着线索定位错误，并把它完美修复的闭环过程。调试四步骤（发现问题 ➔ 找到原因 ➔ 修改 ➔ 再试）与记录日志的重要性。
* **动作规范**：全程使用旁白（不露嘴型）叙述，团团和点点不进行对口型说话，仅通过眼神、手势、点头、观察等肢体动作配合画面。

### 01. 故事开场 00:00-00:15

画面：@1。温馨明亮的艺术工作室大全景，高级水彩绘本风。木质的工具台上乱糟糟地摆放着扳手、螺丝刀、五彩齿轮和各种亮晶晶的机械零件。墙边靠着一辆小巧的蓝色自行车，它的前轮轴心看起来有点歪斜，空气中由于损坏正漂浮着淡淡的灰色"逻辑受阻光晕"。团团和点点推门走进来，好奇地四处张望。
台词/旁白：旁白：清晨的阳光洒进小小的机械工作室。今天，团团和点点要在这里变身成厉害的"技术修理工"！别担心，不管是现实里的机器，还是数字世界里的超级电脑，只要生病了，都逃不过我们今天要去寻找的"修复魔法"！

### 02. 今天学什么 00:15-00:30

画面：@2。延续工作室场景。点点快乐地悬浮在半空，胸前的深色屏幕亮起，浮现出一个散发着青蓝色荧光的"故障捕获（Debugger）"图标——一个小放大镜正牢牢锁定着一只发光的小魔虫。团团顺着点点的指引，小手指新奇地指向靠在墙边的小自行车，眼神里写满了求知欲。
台词/旁白：点点：第12课《找bug的小修理工》！今天我们要去解锁计算机世界里最著名的全自动纠错大本领。去把那些藏在暗处、让机器卡壳的"小捣蛋鬼"一个接一个地揪出来！

### 03. 遇到小麻烦：异常状态 00:30-00:45

画面：@3。近景特写。团团认真地蹲在自行车前，轻轻转了转前轮。车轮晃晃悠悠，发出了刺耳的"吱呀"声，前轮和轴心衔接的缝隙处，正笼罩着代表系统报错（Error）的淡淡红光。地上散落着两个松动的小螺丝，方头机器人坐在一旁，屏幕上弹出了一个闪烁着的大问号。团团认真地皱起了小眉头。
台词/旁白：旁白：哎呀，车轮歪了，螺丝也松了，自行车完全动弹不得！在人工智能和计算机的世界里，这种让程序突然停下、或者变得奇奇怪怪的"小错误"，有一个超级响亮的名字，叫做——"Bug"！

### 04. 观察与思考：故障溯源 00:45-01:00

画面：@4。中景。团团用手指指向车轮轴心上那颗松动的螺丝。点点在半空中展开一面半透明的"故障追踪面板"，面板上用一束柔和的蓝色探索光线，精准地把"车轮晃动（现象）"和"螺丝松了（原因）"连接在了一起。小轮子在灯光下静静地等待着被修复。
台词/旁白：旁白：先看一看，再想一想。优秀的修理工和聪明的AI同学一样，在看到自行车坏了（发现问题）之后，不会瞎敲乱打，而是要顺着线索仔细观察，找出螺丝松掉的真正秘密（找到原因）。

### 05. 动手试一试：状态修复波纹 01:00-01:15

画面：@5。中景。团团从工具台上拿起一把精巧的蓝色小扳手，对准松动的螺丝利落地拧了三下。就在螺丝被拧紧的一瞬间，车轮连接处"啪"地一声激发出一圈圈翠绿色的"状态修复波纹"，原本歪斜的轴线和地面重新对齐，变得笔直坚固！团团的表情从严肃瞬间变成了惊喜，重重点头。
台词/旁白：团团（认真拧扳手）：看我的！对准它，拧紧！
点点：太棒了！把松动的螺丝扭紧（修改步骤），车轮立刻挺直了腰杆。这种"找到Bug、动手改正、让它重新跑起来"的神奇过程，在计算机科学里，就叫做——"调试"！

### 06. 规则出现：电脑里的小飞蛾 01:15-01:30

画面：@6。半空中升起一面巨大的、散发着柔和金光的算法公式面板。面板上写着大字："异常Bug（小错误） ➔ 故障调试（定位并修复）"。面板的角落里，还调皮地画着一只正在扇动翅膀的水彩卡通小飞蛾。团团开心地拍起手来，点点屏幕上的亮黄色灯泡图标璀璨闪烁。
台词/旁白：点点：听说很久很久以前，世界上第一台超级电脑突然不工作了，科学家叔叔们检查了半天，竟然从机器的缝隙里夹出了一只真正的小飞蛾！从那以后，大家就把电脑里的小错误统称为"Bug（虫子）"，把找错误叫做"调试（抓虫子）"啦！

每镜只保留一个主要动作，优先轻推近、横移、角色观察、图标弹出、知识卡点亮和柔和信息流。生成时引用团团、点点及本课新增角色图，保持形象一致。知识文字尽量短，建议后期叠字幕，避免乱码。旁白段落中团团和点点不要张嘴说话，只做观察、点头、指向、摆卡片等非对口型动作。
